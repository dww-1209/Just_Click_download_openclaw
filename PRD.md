# 产品需求文档：OpenClaw 一键安装启动器 - V1.0（一个完美的文档，可以让vibecoding 事半功倍）

## 1. 综述 (Overview)

### 1.1 项目背景与核心问题

OpenClaw 的官方使用方式依赖命令行下载与配置。对于熟悉终端的用户，这种方式可接受；但对于不熟悉命令行的普通用户，它带来了明显的使用门槛：

- 不知道该输入什么命令
- 不清楚安装过程中的每一步含义
- 容易因为路径、权限、网络、配置参数出错而中断
- 安装完成后仍需手动配置，无法直接进入可用状态

本产品的目标，是提供一个类似 Claw X 体验的桌面端程序，把 OpenClaw 的“下载、安装、默认配置、启动 WebChat”封装成一个傻瓜式流程。用户只需要按 Next，尽量不接触命令行，就能完成安装和使用。

本产品的核心原则是：

- 面向小白用户
- 图形化引导
- 默认配置优先
- 不暴露复杂命令
- 命令行仅在后台静默执行
- 安装完成后直接进入 WebChat

### 1.2 产品定位

本产品是一个跨平台桌面安装启动器，不是 OpenClaw 本身的替代品。它的职责是：

- 调用操作系统能力
- 静默执行官方提供的命令行安装方式
- 自动完成默认配置
- 启动 WebChat 入口

### 1.3 目标用户

- 想使用 OpenClaw，但不会使用命令行的普通用户
- 希望快速体验 OpenClaw 的新手用户
- 希望减少安装成本和学习成本的用户

### 1.4 核心业务流程 / 用户旅程地图

1. 启动与欢迎阶段（US-01） - 展示产品简介，用户点击 Next 进入流程
2. 环境检测阶段（US-02） - 自动检查系统、网络、权限与磁盘状态
3. 安装路径选择阶段（US-03） - 用户确认默认路径或选择自定义路径
4. 下载与安装阶段（US-04） - 后台静默调用系统命令执行官方下载与安装命令
5. 默认配置与启动 WebChat 阶段（US-05） - 自动写入默认配置并直接打开 WebChat

### 1.5 全局数据契约表

以下数据契约在所有故事中共享，保证跨阶段一致性。

| 名称 | 类型 | 说明 | 来源故事 | 备注 |
|---|---|---|---|---|
| `stage` | enum | 当前安装阶段 | 全局 | `welcome / env_check / path_select / installing / configuring / completed / error` |
| `installStatus` | enum | 安装任务状态 | 全局 | `idle / running / success / failed / cancelled` |
| `checkStatus` | enum | 环境检测状态 | US-02 | `ok / warning / failed` |
| `pathSource` | enum | 安装路径来源 | US-03 | `default / custom` |
| `osType` | enum | 操作系统类型 | US-02 / US-04 | `windows / macos / linux` |
| `installPath` | string | 用户最终选择的安装路径 | US-03 | 必填 |
| `officialCommandTemplate` | string | 官方命令模板（按系统区分） | US-04 | 由配置文件或版本表维护 |
| `shellMode` | enum | 命令执行方式 | US-04 | `hidden / visible`，本产品固定为 `hidden` |
| `configMode` | enum | 配置模式 | US-05 | 固定为 `default` |
| `serviceHost` | string | Web 服务地址主机 | US-05 | 默认 `localhost` |
| `servicePort` | number | Web 服务端口 | US-05 | 默认端口由产品定义 |
| `webchatUrl` | string | WebChat 访问地址 | US-05 | 由服务启动结果生成 |
| `logLines` | array[string] | 安装与配置日志 | US-04 / US-05 | 仅用于界面展示与排障 |
| `errorCode` | string | 错误码 | 全局 | 仅用于内部排障，不对小白暴露过多术语 |

# 阶段一：启动与欢迎

---

## US-01：作为首次用户，我希望打开安装程序后看到一个清晰的欢迎页面，并通过点击 Next 开始安装流程，以便我可以用最简单的方式进入 OpenClaw 安装。

### 价值陈述 (Value Statement)

- 作为：首次使用 OpenClaw 的普通用户
- 我希望：打开程序后看到一个简单清晰的欢迎页，并通过点击 Next 进入下一步
- 以便于：我不需要理解命令行就可以开始安装

### 业务规则与逻辑 (Business Logic)

1. 前置条件
   - 用户已下载并双击启动本安装程序
   - 程序可在 Windows、macOS、Linux 上运行

2. 操作流程 (Happy Path)
   1. 用户启动程序
   2. 系统展示欢迎页面
   3. 页面展示产品名称和一句简介
   4. 用户点击 Next
   5. 系统进入环境检测阶段（US-02）

3. 异常处理 (Error Handling)
   - 程序启动失败：提示“程序无法启动，请重新下载或检查系统兼容性”
   - 权限不足：提示“建议以管理员身份运行”
   - 页面渲染失败：提供“重试”按钮

4. 性能与容量提示
   - 首屏加载时间应尽量小于 2 秒
   - 点击 Next 后页面切换应快速响应
   - 不涉及大量数据加载

### 验收标准 (Acceptance Criteria)

- 场景1：正常进入欢迎页
  - GIVEN 用户双击启动程序
  - WHEN 程序成功加载
  - THEN 显示欢迎页、产品简介、Next 按钮和退出按钮

- 场景2：点击 Next 进入下一步
  - GIVEN 用户处于欢迎页
  - WHEN 用户点击 Next
  - THEN 系统进入环境检测阶段

- 场景3：点击退出
  - GIVEN 用户处于欢迎页
  - WHEN 用户点击退出
  - THEN 程序关闭并终止后续流程

### 技术实现概要 (Technical Implementation Brief)

- 影响范围：桌面端安装程序主窗口
- 前端 / 客户端：
  - 显示产品名称、简介、Next、退出按钮
  - Next 仅负责进入下一步
  - 退出按钮关闭应用
- 后端：
  - 本阶段无远程后端
- 数据库 / 存储：
  - 本阶段不需要数据库
  - 可记录本地日志用于排障

### 数据模型 / 接口契约 (Data Contracts & APIs)

- 本阶段无外部 API
- 内部状态：
  - `stage = welcome`
  - `installStatus = idle`

### 监控与运营 (Observability & Operations)

- 记录本地事件：
  - `installer_launch`
  - `welcome_page_view`
  - `welcome_next_clicked`
  - `installer_exit`

### 约束与边界 (Constraints & Boundaries)

- 不展示复杂技术信息
- 不要求用户填写信息
- 不在此阶段启动任何安装或下载动作

### 非功能性需求 (Non-Functional)

- 界面简洁
- 启动稳定
- 支持键盘操作（例如 Enter 触发 Next）
- 三种操作系统上的体验尽量一致

### 开放问题与待确认事项 (Open Questions & Follow-ups)

- 是否需要品牌 Logo 的正式定稿
- 是否需要多语言支持
- 是否需要在欢迎页增加许可提示

# 阶段二：环境检测

---

## US-02：作为用户，我希望系统自动检测我的设备环境，并在必要时提示问题，以便顺利完成 OpenClaw 安装。

### 价值陈述 (Value Statement)

- 作为：普通用户
- 我希望：程序自动检查我的环境状态
- 以便于：提前发现问题，避免安装失败

### 业务规则与逻辑 (Business Logic)

1. 前置条件
   - 用户已从 US-01 点击 Next
   - 程序进入环境检测阶段

2. 需要检测的内容
   - 操作系统类型
   - 磁盘空间是否足够
   - 网络是否可用
   - 当前用户权限是否足够

3. 操作流程 (Happy Path)
   1. 页面自动开始检测
   2. 系统逐项检查环境
   3. 页面显示检测中和各项结果
   4. 所有必要条件满足后，允许用户进入下一步

4. 异常处理 (Error Handling)
   - 磁盘空间不足：阻止继续，并提示用户释放空间
   - 网络不可用：提示重新连接网络，并提供重试
   - 权限不足：提示风险，但不立即阻断（除非后续安装阶段失败）
   - 系统不兼容：阻止继续

5. 性能与容量提示
   - 环境检测尽量在 3 秒内完成
   - 检测过程不能卡住页面
   - 网络检测允许短超时后重试

### 验收标准 (Acceptance Criteria)

- 场景1：环境检测通过
  - GIVEN 用户进入环境检测页
  - WHEN 系统完成检测且条件满足
  - THEN 页面显示通过状态，并允许进入下一步

- 场景2：磁盘不足
  - GIVEN 磁盘空间不足
  - WHEN 系统完成检测
  - THEN 页面提示空间不足并阻止继续

- 场景3：网络异常
  - GIVEN 网络不可用
  - WHEN 系统检测网络
  - THEN 显示错误并提供重试按钮

- 场景4：权限不足
  - GIVEN 当前权限较低
  - WHEN 系统检测权限
  - THEN 显示警告但仍允许用户继续到下一步（由后续安装阶段再次验证）

### 技术实现概要 (Technical Implementation Brief)

- 影响范围：环境检测模块、安装向导页面
- 前端 / 客户端：
  - 展示检测进度
  - 展示每项检测结果
  - 控制 Next 按钮是否可用
- 后端：
  - 无远程服务
  - 通过本地系统 API 读取系统信息、磁盘信息和网络连通性
- 数据库 / 存储：
  - 可写入本地检测日志

# 阶段三：安装路径选择

---

## US-03：作为用户，我希望可以选择或确认安装路径，以便在合适的位置安装 OpenClaw。

### 价值陈述 (Value Statement)

- 作为：普通用户
- 我希望：可以保持默认安装路径，或者自己选择一个路径
- 以便于：更方便地管理磁盘空间和安装位置

### 业务规则与逻辑 (Business Logic)

1. 前置条件
   - 已完成 US-02
   - 系统允许继续安装

2. 默认路径策略
   - Windows：`C:\Program Files\OpenClaw`
   - macOS：`/Applications/OpenClaw`
   - Linux：`/usr/local/OpenClaw`
   - 若默认路径不可写，则提示用户选择其他路径

3. 操作流程 (Happy Path)
   1. 页面展示默认路径
   2. 用户可直接点击 Next
   3. 或点击“浏览”选择自定义路径
   4. 系统检查路径合法性
   5. 校验通过后进入下载与安装阶段

4. 异常处理 (Error Handling)
   - 路径不可写：提示更换路径或以管理员身份运行
   - 路径不存在：尝试自动创建，失败则提示
   - 磁盘空间不足：阻止继续

5. 性能与容量提示
   - 路径校验应快速完成
   - 文件系统操作不得冻结界面
   - 支持跨平台路径处理

### 验收标准 (Acceptance Criteria)

- 场景1：使用默认路径
  - GIVEN 用户进入安装路径页
  - WHEN 用户不修改路径并点击 Next
  - THEN 系统使用默认路径进入下一步

- 场景2：使用自定义路径
  - GIVEN 用户点击浏览并选择路径
  - WHEN 系统校验通过
  - THEN 进入下一步

- 场景3：路径无权限
  - GIVEN 用户选择了不可写路径
  - WHEN 系统校验路径
  - THEN 显示错误并阻止继续

- 场景4：路径冲突
  - GIVEN 目标路径中已存在 OpenClaw
  - WHEN 用户继续
  - THEN 提示覆盖或更换路径

### 技术实现概要 (Technical Implementation Brief)

- 影响范围：安装路径选择页、文件系统校验模块
- 前端 / 客户端：
  - 路径输入框
  - “浏览”按钮
  - 提示信息区域
  - Next 按钮
- 后端：
  - 本地路径校验
  - 目录创建检测
  - 可写性检查
- 数据库 / 存储：
  - 可写入本地安装配置草稿

  - `install_path_view`
  - `install_path_changed`
  - `install_path_invalid`
  - `install_path_confirmed`

### 约束与边界 (Constraints & Boundaries)

- 不允许非法路径字符
- 不建议使用系统关键目录
- 不支持网络盘作为默认方案
- 默认路径应符合各平台规范

### 非功能性需求 (Non-Functional)

- 路径选择和校验要及时反馈
- 错误提示必须简单明确

# 阶段四：下载与安装

---

## US-04：作为用户，我希望程序在后台静默调用系统命令执行官方提供的下载与安装命令，并展示安装状态，以便我无需接触命令行也能完成安装。

### 价值陈述 (Value Statement)

- 作为：不熟悉命令行的用户
- 我希望：程序在后台帮我执行官方安装命令
- 以便于：我看不到命令行窗口，也不需要手动输入命令

### 业务规则与逻辑 (Business Logic)

1. 前置条件
   - 已完成 US-01、US-02、US-03
   - 网络可用
   - 安装路径合法

2. 核心流程 (Happy Path)
   1. 用户点击 Next 后进入安装阶段
   2. 程序根据操作系统生成对应的官方命令模板
   3. 程序以隐藏方式启动系统 Shell
   4. 系统静默执行官方安装命令
   5. 程序捕获输出并更新界面状态
   6. 安装成功后进入 US-05

3. 关键约束
   - 不弹出可见命令行窗口
   - 不让用户手动输入命令
   - 仅执行官方命令，不执行任意用户命令
   - 后台执行必须可控、可追踪

4. 异常处理 (Error Handling)
   - 下载失败：提示网络问题并提供重试
   - 命令执行失败：展示简化错误信息和日志
   - 权限问题：提示使用管理员身份重新运行
   - 用户中断：安全终止流程并清理临时文件

5. 性能与容量提示
   - UI 不得卡死
   - 长任务需要持续反馈
   - 日志输出应尽量实时
   - 安装时长可能较长，界面必须保持响应

### 验收标准 (Acceptance Criteria)

- 场景1：正常安装
  - GIVEN 用户已完成安装路径选择
  - WHEN 用户点击 Next
  - THEN 程序在后台静默执行官方安装命令
  - AND UI 显示安装状态
  - AND 最终提示安装成功

- 场景2：网络中断
  - GIVEN 安装过程中的网络不可用
  - WHEN 下载失败
  - THEN 页面提示网络错误并提供重试

- 场景3：命令执行失败
  - GIVEN 官方命令返回错误
  - WHEN 程序捕获失败

# 阶段五：默认配置与启动 WebChat

---

## US-05：作为用户，我希望在安装完成后系统自动完成默认配置，并直接打开 WebChat，以便我无需手动配置即可立即使用 OpenClaw。

### 价值陈述 (Value Statement)

- 作为：刚安装完成的用户
- 我希望：程序自动完成默认配置并直接进入 WebChat
- 以便于：我不需要再手动配置任何参数

### 业务规则与逻辑 (Business Logic)

1. 前置条件
   - US-04 成功完成
   - OpenClaw 已安装成功
   - 系统可正常启动本地服务

2. 默认配置内容
   - 自动生成默认配置文件
   - 使用默认端口和默认主机
   - 启动必要服务
   - 不弹出高级设置页面
   - 不要求用户填写任何配置项

3. 操作流程 (Happy Path)
   1. 安装完成后自动进入配置阶段
   2. 程序写入默认配置
   3. 程序启动 OpenClaw 服务
   4. 程序执行健康检查
   5. 服务启动成功后自动打开 WebChat

4. 启动 WebChat 的方式
   - MVP 推荐使用系统默认浏览器打开本地地址
   - 默认地址由启动结果生成
   - 若浏览器无法自动打开，则提供可复制的访问链接

5. 异常处理 (Error Handling)
   - 配置写入失败：提示重试并记录日志
   - 服务启动失败：提示端口冲突或依赖问题
   - WebChat 打不开：展示手动访问地址
   - 健康检查失败：提示服务未就绪

6. 性能与容量提示
   - 默认配置过程应尽量快速
   - 健康检查结果应尽快反馈
   - WebChat 打开动作应在服务就绪后触发

### 验收标准 (Acceptance Criteria)

- 场景1：正常自动配置
  - GIVEN OpenClaw 安装完成
  - WHEN 系统进入配置阶段

  - GIVEN 配置文件写入失败

- 场景3：服务启动失败

  - THEN 显示失败原因并提供可执行提示

- 场景4：浏览器未自动打开
  - GIVEN 服务启动成功但浏览器未打开
  - WHEN 页面进入完成状态
  - THEN 展示可复制的 WebChat 地址
